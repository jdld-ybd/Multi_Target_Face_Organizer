# 🤝 多目标人脸分类整理器 - 贡献指南

感谢你愿意为该项目贡献代码/文档！本指南将告诉你如何正确参与项目贡献，确保协作高效、代码规范，让项目更完善。

## 📌 贡献方式

你可以通过以下任意一种方式参与贡献：

1. **提交Bug反馈**：发现程序运行异常、安装失败、匹配错误等问题，提交Issue
2. **提出功能建议**：如支持PNG/WEBP格式、批量导入目标人脸、可视化界面等，提交Issue讨论
3. **代码贡献**：修复已知Bug、实现新功能、优化代码逻辑，提交Pull Request（PR）
4. **文档完善**：修正README错别字、补充使用教程、优化安装步骤等
5. **测试验证**：在不同系统/Python版本下测试程序，反馈兼容问题

## 🐛 提交Bug报告

### 提Bug前请先确认

1. 已阅读 [README.md](./README.md) 的「常见问题」，未找到解决方案
2. 已使用项目指定的Python版本（3.6~3.11）和依赖版本（见requirements.txt）
3. 已排查自身环境问题（如CMake未配置、目标人脸图片无效等）

### 提交Issue要求

**必须使用仓库的「Bug报告模板」**，按模板填写以下信息：

- **Bug现象**：清晰描述程序的异常表现（如安装报错、运行中断、匹配错误）
- **复现步骤**：一步步描述操作，让开发者能快速复现
- **环境信息**：系统、Python版本、安装方式（pip/Anaconda）
- **错误日志**：粘贴完整的报错信息（文本优先）
- **补充信息**：目标人脸图片类型、待处理图片格式（可选）

## ✨ 提出功能建议

### 提交Issue要求

**必须使用仓库的「功能请求模板」**，按模板填写：

- **功能描述**：清晰说明想要的功能（如"支持PNG图片处理""添加匹配阈值命令行参数"）

## 🔧 提交Pull Request（PR）

### 前提条件

1. 已Fork本仓库到你的GitHub账号
2. 本地搭建好项目开发环境（见README.md的「快速开始」）
3. 已阅读项目代码，了解核心逻辑（main.py的三大核心函数）

### PR提交前准备

1. **关联Issue**：本次PR需对应一个已提交的Issue（Bug修复/功能建议），未提Issue的PR将被驳回
2. **代码规范**：遵循 **Python PEP8** 规范（缩进4个空格、变量名小写下划线、函数名小写下划线）
3. **功能测试**：本地运行修改后的代码，确保原有功能正常运行
4. **无冗余代码**：删除调试代码（如print/debug）、无用注释，保持代码整洁
5. **版本兼容**：确保修改后的代码在Python 3.6~3.11下均可运行

### PR提交流程

1. **创建分支**：Fork本仓库后，新建分支（禁止直接在main/master分支开发）：
   ```bash
   # 修复Bug用bugfix/前缀
   git checkout -b bugfix/fix-target-files-duplicate
   # 新增功能用feature/前缀
   git checkout -b feature/support-png-format
   ```

2. **提交修改**：本地修改代码/文档，提交时写清晰的提交信息：
   ```bash
   # Bug修复示例
   git commit -m "fix: 修复main.py中target_files重复加载的问题"
   # 功能新增示例
   git commit -m "feat: 新增PNG图片格式支持，修改get_jpg_files函数"
   # 文档完善示例
   git commit -m "docs: 补充README中Anaconda安装的详细步骤"
   ```

3. **推送分支**：将分支推送到你的Fork仓库：
   ```bash
   git push origin bugfix/fix-target-files-duplicate
   ```

4. **创建PR**：到本项目GitHub页面提交PR，填写PR描述（参考PR模板）

5. **等待审核**：维护者将尽快审核，按反馈调整后重新提交

## 📝 代码规范

### 基础规范

- 遵循 **Python PEP8** 规范，行宽不超过120字符
- 缩进使用4个空格，禁止使用制表符
- 变量/函数名：小写字母 + 下划线（如`target_encodings`/`load_target_encodings`）
- 类名：驼峰命名法（如`FaceOrganizerApp`）
- 常量：全大写字母 + 下划线（如`TARGET_FOLDER`）

### 代码结构

- **配置参数**：放在main.py顶部，单独注释，便于用户修改
- **函数注释**：新增/修改的函数需添加文档字符串，说明功能、参数、返回值
- **异常处理**：对文件操作/图片加载添加try-except，避免程序中断
- **路径处理**：使用os.path模块，保证跨系统兼容，禁止硬编码路径
- **模块化**：将相关功能组织成函数，保持代码结构清晰
